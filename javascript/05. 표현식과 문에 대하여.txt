>값이란?

- 표현식이 평가되어 생성된 결과를 말한다
잉?? 이게 무슨 말이야?? 평가가 뭔데?? => 식을 해석해서 '값을 생성'하거나 '참조'하는 것을
말한데~  (참조란? 참고로 비교하여 대조하여 보는 것)

>값의 특징

모든 값은 데이터 타입을 가지고 비트의 나열로 저장된다고 한다~ 
그리고 데이터 타입에 따라 다르게 해석이 된다~ 
ex) 0100 0001  숫자로 해석하면 65 영어로 해석하면 A 

변수는 하나의 값을 저장하기 위해 확보한 메모리 공간 자체 또는 그 메모리 공간을
식별하기 위해 붙인 이름이라고 했다. 
		
	=>따라서 변수에 할당되는 것은 값이다.

>리터럴에 대하여 

리터럴은 
사람이 이해할 수있는 문자/약속된 기호를 사용하여 값을 생성해내는 표기법을 뜻해요~

예를 들어 3 3은 아라비아 숫자 이기도 하고 숫자 리터럴이기도 합니다~

3을 코드에 기술하면 자바스크립트 엔진은 이를 평가해 숫자 값 3을 생성해냅니다

>리터럴의 종류 

정수, 부동소수점. 2진수, 8진수, 16진수, 문자열,불리언, null, undefined, 객체. 배열, 함수, 정규
표현식 리터럴이 있습니다~


>표현식에 대하여

표현식은 값으로 평가될 수 있는 문 입니다. 

즉 표현식이 평가되면 새로운 값을 생성하거나 기존 값을 참조 합니다


값으로 평가될 수있는 것은 모오오오오두 표현식이다!

>문 에 대해서 

프로그램을 구성하는 기본 단위이자 최소 실행 단위!

종류는 선언문, 할당문, 조건문, 반복문 등이 있다 

> 세미 콜론과 세미콜론 자동 삽입 기능


세미콜론을 입력하는 이유는 문의 종료를 나타내기 위해서 이다 
자바스크립트에서는 세미콜론을 잘못해서 못 찍어도 세미콜론 자동 삽입 기능(ASI: automatic semicolon insertion)으로
입력하지 않아도 문의 종료 지점을 계산해서 찍어 준다고 함!!(매우 친절하다 자바스크립트)
-하지만 세미콜론을 권장하는 분위기이다...ㅎㅎ


> 표현식인 문과 표현식이 아닌 문에 대하여

표현식인 문?? 표현식이 아닌 문?? 이걸 왜 구별해야하는진 모르겠지만
일단 알아본 결과론

표현식인 문은 var 변수에 할당해 봤을때 괜찮고
표현식이 아닌 문은 할당 했을때 오류가 뜨는 것 이라고 한다


예시로는 var x라는 문을 var not에 할당 한다고 생각해보자

var not = var x;     보기만 해도 뭔가 잘못된것 같아 보인다.... 오류! 이것을
표현식이 아닌 문이라고 한다  
하지만 
var not = x = 100 이라했을 때는 괜찮다???

맞다 x = 100은 표현식이다 

이런식으로 표현식인 문과 아닌것을 구별하는 법을 배웠다!!!!
